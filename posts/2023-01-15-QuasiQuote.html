Show 인스턴스

일단 항상 빈 문자열을 돌려주는 Show 인스턴스를 만드는 걸 보겠습니다. - leonidas의 튜토리얼의 번역입니다.

splice에 쓸 emptyShow를 정의합니다.

{-# LANGUAGE TemplateHaskell, FlexibleInstances #-}
module CustomShow where
import Language.Haskell.TH

emptyShow :: Name -> Q [Dec]
emptyShow name = [d|instance Show $(conT name) where show _ = ""|]

TH가 splice를 instance 코드로 대체합니다.

{-# LANGUAGE TemplateHaskell #-}
import CustomShow

data MyData = MyData
    { foo :: String
    , bar :: Int
    }

-- $(emptyShow ''MyData)와 같습니다. 탑 레벨에서는 $(...)을 생략할 수 있습니다.
emptyShow ''MyData

main = print $ MyData { foo = "bar", bar = 5 }

두 개의 홑 따옴표quotes가 있는 ''MyData는 mkName "MyData"와 같습니다.
※ 위와 같이 emptyShow 함수는 다른 파일에 정의되어 있어야 합니다. 이유는, splice에서 쓸 때 완전히 컴파일된 상태여야 해서 그렇습니다.

빈 문자열을 출력하게는 만들었으니, 이제 여기에 MyData같은 레코드의 필드값을 가져와서 실제 레코드의 Show 인스턴스를 만들겠습니다.

listFields :: Name -> Q [Dec]
listFields name = do
    TyConI (DataD _ _ _ [RecC _ fields] _) <- reify name

reify가 돌려주는 값은 복잡한데, 여기선 레코드 타입 생성자를 하나만 갖는 것으로 하드 코딩해서 패턴 매칭합니다. 이렇게 하면 필드들을 [RecC _ fields]로 받아 올 수 있습니다. fields에는 (name, strict, type) 튜플이 들어 있습니다. 이 중에서 name만 따로 names 목록을 만들겠습니다.

    let names = map (\(name,_,_) -> name) fields

이제 레코드의 필드도 출력하는 함수를 위한 quotation을 만들 수 있습니다.

let showField :: Name -> Q Exp
    showField name = [| \x -> s ++ " = " ++ show ($(varE name) x) |] where
        s = nameBase name

필드를 넘겨주면 "name = value" 문자열을 만듭니다. quotation 안에서 $(varE name)을 빼면 일반 하스켈 코드입니다. quotation안에 또 splice를 둘 수 있습니다.
showFiled 함수를 모든 필드에 적용하면,

let showFields :: Q Exp
    showFields = listE $ map showField names

이를 써서 Show 인스턴스를 선언하면

[d|instance Show $(conT name) where
    show x = intercalate ", " (map ($ x) $showFields)|]

이제 GHC가 템플릿을 이용해 MyData의 Show 인스턴스를 만들어 냅니다. 그럼 아래 같이 쓸 수 있게 됩니다.

 print $ MyData { foo = "bar", bar = 5 }
foo = "bar", bar = 5

하스켈 코드 표현을 위한 ADT

TH에서는 보통의 algebraic data types로 하스켈 프로그램을 표현합니다. 하스켈 언어 문법에 맞게 모델링하고, 하스켈 코드에 대응하는 AST(Abstract Syntax Tree)로 다음과 같이 표현합니다.

하스켈 Expression  은 Exp 타입으로,
하스켈 Pattern     은 Pat 타입으로,
하스켈 Literal     은 Lit 타입으로,
하스켈 Declaration 은 Dec 타입으로 표현한다.

varx = VarE (mkName "x")는 Expression x
patx = VarP (mkName "x")는 Pattern x
str = LitE (StringL "str")는 constant expression "str"
tuple =TupE [varx, str]는 Expression (x, "str")
LamE[patx] 튜플은 람다식 (\x -> (x, "str"))

Expressoin인 생성자들은 모두 ~E가 붙어 있고, Pattern인 생성자들은 모두 ~P가 붙어 있습니다.
mkName함수는 함수명이나 타입명 같은 식별자Identifier들에 대응하는 값을 생성합니다.

표현식 quote는 옥스포드 브라켓([|...|])으로 감싸져 있습니다.

[| 표현식 |] 또는 [e| 표현식 |] quotes는 Quote m => m Exp 타입
[d| 표현식 |] quotes는 Quote m => m [Dec] 타입
[t| 표현식 |] quotes는 Quote m => m Type 타입
[p| 표현식 |] quotes는 Quote m => m Pat 타입

Q(Quotation) 모나드

newtype Q a = Q (Env -> IO a)

논문에는 이렇게 강화된augmented IO쯤으로 설명하고, 실제 GHC에는 다음과 같이 정의되어 있습니다.

class (MonadIO m, MonadFail m) => Quasi m where
  qNewName :: String -> m Name
  qReport  :: Bool -> String -> m ()
  qRecover :: ... 
  ...
newtype Q a = Q { unQ :: forall m. Quasi m => m a }

Q 모나드의 바인드는 아래와 같이 정의되어 있습니다.

instance Monad Q where
  Q m >>= k  = Q (m >>= \x -> unQ (k x))

Q 모나드가 추가로 끌고 다니는 Env에는 아래 요소가 들어가 있습니다.

  1. Fresh name generation
  2. program reification
  3. error reporting
