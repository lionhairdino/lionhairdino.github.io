확장 ExistentialQuantification

Posted on June 12, 2020

원래 수학에서 쓰이는 뜻은 조건을 만족하는 모든 값 forall 조건을 만족하는 최소 하나 이상의 값 exists x

ScopedTypeVariables RankNTypes/Rank2Types ExistentialQuantificatio

forall 이 쓰이는 경우는 세 가지 확장을 켰을 때다. 여기서는 data 선언할 때 보이는 forall 만 다룬다.

이 확장을 켜면, 여러 타입들을 뭉뚱그려 하나의 타입으로 취급할 수 있다. ~~~ haskell {-# LANGUAGE ExistentialQuantification #-} data T’ = forall a. Show a => MkT’ a

heteroList’ = [MkT’ 5, MkT’ (), MkT’ True, MkT’ “Sartre”] main = mapM_ ((MkT’ x) -> print x) heteroList’ ~~~ 위 소스에서 forall 을 빼면

error: Not in scope: type variable ‘a’
2 | data T' = Show a => MkT' a

이런 오류가 난다.

forall 을 쓰면 5, (), True, “Sartre” 를 모두 다룰 수 있는 MkT’ 이라는 타입을 만들 수 있다. 모든 타입을 다 받아들이게 하면 실용적인 코드로는 별 쓸 일이 없다. 하지만 여기에 Show a 라는 제약(한정자라 번역할까?)을 주면 모든 타입이 아닌, show 함수를 적용할 수 있는 타입으로 대상이 축소 된다. 이런식으로 쓰면 실용 코드로 쓰일 수 있다.

뭔가 동적으로 타입이 결정되는 것 같다. 타입을 받는 곳에 “a” 를 넘기면 [Char] 타입이 되고, ‘a’ 를 넘기면 Char 타입이 된다. 나중에 어떤 값이 들어가냐에 따라 동적으로 타입이 결정될 필요가 있을때도 쓰일 수 있겠다.

existentially quantified 란 말은, 동적으로만 초기화 될 수 있는 타입 변수를 가지고 있다와 갈은 말이라고 한다. 무슨 말일까?

영단어 뜻 existentially 실체론적으로, 실존(주의)적으로 quantified 뭔가를 울타리 안에 가두는, 한정, 제약 느낌. 타입을 정할 순 없지만, 타입 클래스를 한정할 순 있다.

data S = forall a. Show a => S a

a 가 무슨 타입이라고 정한다는 말이 아니라, a 는 Show 의 인스턴스여야 한다는 말이다. 타입이 속한 클래스를 정해서 타입을 한정 한다. 이걸 GADT 문법으로 표현하면,

{-# LANGUAGE GADTs #-}
data S where
    S :: Show a => a -> S
