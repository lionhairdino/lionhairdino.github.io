ST 모나드 - forall의 강력함 (작성 예정)

Posted on December 12, 2022

IO와 비슷하지만, forall의 추가로 타입 체커가 많은 걸 보장합니다.

forall

forall - 아무거나와 모든 것의 차이

준비 운동

RankNType

https://stackoverflow.com/questions/45287954/is-having-a-a-b-b-equivalent-to-having-an-a 답변에서 소스 발췌

forall b. (a -> b) -> b

이 타입 함수는 (a -> b) 함수를 받으면 b를 돌려줍니다. 안에 a를 가지고 있어야 합니다. 하지만 a가 뭔지를 모르니 구현이 안되겠구나로 생각했습니다. 그런데, a가 당장은 뭔지 모르지만, 나중에 알려 줄 방법이 있다면 어떻게 될까요?
보기 좋게 하기 위해, 위 서명으로 타입을 만들고,

-- 여기서 a는 forall이 아니라, 나중에 Box a 타입이 만들어질 때 결정됩니다.
type Box a = forall b. (a -> b) -> b

이 타입을 만들어내는 함수를 따로 두면 됩니다.

box :: a -> Box a
box a = \f -> f a

forall b. (a -> b) -> b 타입을 말로 읽어 보았습니다.
예를 들어 10이상이면 big, 아래면 small이라 하고, 이를 구분하는 함수를 isBig이라 하면, “5라는 숫자는 isBig을 만나면 small이 될 수” 입니다. 즉,

“a는 (a->b)를 만나면 b가 될 놈” 입니다.

그래서 a는 forall b. (a -> b) -> b로 표현할 수 있다는 얘기입니다.

폴리모픽 자리에는 언젠가는 구체 타입이 들어온다.

forall a. Show a => a라고 되어 있으면, 언젠가 Show 인스턴스가 정의되어 있는 타입이 들어올테고,
forall a. a라고 되어 있으면, 제한 없이 어떤 타입이든 들어올 겁니다.
(forall a. a -> b)라고 되어 있으면, “모든 타입을 받아도 문제 없이 b를 내뱉는 함수”가 들어 옵니다.

이 제한을 따르는 나중에 들어올 값에, 문제가 생기지 않게 구현되어 있다는 얘기입니다. 서명은 함수가 어떻게 구현되어 있는지 알리는 역할을 합니다. 제한된 타입들이 공통적으로 갖고 있는 메소드만을 써야, 해당 타입들이 들어왔을 때 문제가 생기지 않습니다. 모든 타입이 공통적으로 가지고 있는 메소드는 없습니다. forall a 즉 모든 타입에 문제가 생기지 않으려면, a 타입에 의존하는 동작이 아예 없어야 합니다. 아니면, 모든 타입에는 bottom이 있으니, undefined로 구현하는 방법이 있을 순 있습니다.

Return Type Polymorphic

Return 타입은 들어오는 값에 따라 결정 됩니다.

ST모나드와 forall

-- 암시적으로 들어 있는 forall을 같이 써주면 
-- forall a. (forall s. ST s a) -> a 
runsST :: (forall s. ST s a) -> a

ST안에서 s에 의존하는 값을 만들어 이리 저리 사용하다가 나중에 이 값을 꺼내려고 runST를 돌린다고 가정하면,

runST :: forall a. (forall s. ST s (s에 의존하는 a값)) -> (s에 의존하는 a값)

이 타입이 말이 돼야 하는데, 매개 변수 (forall s. ST s (s에 의존하는 a값)) 안에 들어있는 s와 결과값 (s에 의존하는 a값)의 s를 같게 만들 방법이 없습니다. forall의 위치를 잘 보시기 바랍니다. RankN 상황입니다.

생각을 안에 있던 데이터를 직접 꺼내오는 작업으로 바로 연결하지 말고, 작업을 하기 전 컴파일러 입장에서 타입이 맞는지 결정을 내릴 수 있는지 없는지를 생각해 보시기 바랍니다.

The forall ensures that the internal state used by the ST computation is inaccessible to the rest of the program.

즉 타입만으로 ST안에서 만든 Ref가 ST를 벗어날 수 없음을 보장할 수 있습니다.

newtype ST s a = ST (STRep s a) type STRep s a = State# s -> (# State# s, a #)

runST :: (forall s. ST s a) -> a
